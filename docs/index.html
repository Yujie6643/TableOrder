SORA then keeps Row-major as a shared branch and sparsely selects complementary routed orders for each token and attention head.

Structure-Preserving Table Blocking

The table is first separated into header rows and body rows. The body is then partitioned using predefined block_rows and block_cols. Each local block retains the corresponding header metadata, block identifier, and row information before all blocks are concatenated in their original order.

Order-aware Attention for a Single Cell Order

Each order maps the same two-dimensional coordinates to a different sequence of one-dimensional positions. Standard RoPE injects these order-induced positions into the query and key. Cell semantics stay unchanged, and no additional learnable positional parameters are introduced.

Sparse Order Routing Attention

Row-major is the fixed shared order and always provides the base attention output. Column, Hilbert, Spiral, and Snake are the routed orders. For each token and attention head, the router scores these candidates and sparsely selects the Top-K routed views.

Curriculum for Two-stage Order Learning

During the first N epochs, the model uses only standard Row-major to establish a stable table-understanding prior. From epoch N onward, routed orders and sparse gating are enabled, while controlled randomness encourages exploration.
